# Governance

Oikoscope is an open source project stewarded by named maintainers and aligned with the [CHAOSS Data Science Working Group](https://chaoss.community/inside-the-chaoss-data-science-working-group/).

## Maintainers

Maintainers are listed in [`README.md`](README.md). They merge contributions that meet project standards, cut releases, and escalate disputes or scope questions to the working group when useful.

## Decision-making

- **Day-to-day**: Issues and pull requests are the primary forum. Maintainers seek consensus with contributors and may use lazy consensus when feedback has been solicited and no substantive objection remains.
- **Direction and scope**: Material changes to project goals or inclusion criteria are discussed in issues (or Slack `#data-science`) and SHOULD be visible to the wider CHAOSS community when they affect shared metrics or branding.

## Contributions

All contributors agree to the [Developer Certificate of Origin (DCO)](https://developercertificate.org/) via sign-off on commits. See [`CONTRIBUTING.md`](CONTRIBUTING.md).

## Code of Conduct

Participation is governed by the [CHAOSS Community Code of Conduct](https://chaoss.community/code-of-conduct). See [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).
