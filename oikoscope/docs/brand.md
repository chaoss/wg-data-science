# Oikoscope brand

## Canonical naming

- **Project name:** Oikoscope (capital O; one word).
- **Repository (current):** [`github.com/chaoss/wg-data-science`](https://github.com/chaoss/wg-data-science/tree/main/oikoscope) — Oikoscope lives under `oikoscope/` until a dedicated remote is published.
- **Repository (future):** [`github.com/chaoss/oikoscope`](https://github.com/chaoss/oikoscope) — intended standalone home after subtree split; update `pyproject.toml` `[project.urls]` when that goes live.
- **Python distribution / CLI:** `oikoscope` (lowercase on PyPI if published later).
- **Avoid:** “OiKoScope” mixed casing, “Oiko Scope” as two words in titles.

## Tagline (primary)

**Structured, reproducible views of open source foundation and project health, aligned with CHAOSS metrics.**

Use this in READMEs, repo descriptions, and talk titles unless a shorter line is required.

## Short line (bios, social)

**CHAOSS-aligned foundation and project health metadata —** the *oikos* of open source communities, observed transparently. [Historical note on *oikos*.](https://en.wikipedia.org/wiki/Oikos)

## Visual identity

- **Logo:** Horizontal wordmark + icon, `assets/oikoscope-logo.png` (add files under `assets/`; regenerate favicons with `scripts/generate_favicons.py` when the master artwork changes).
- **Alt text:** “Oikoscope”.
- **Favicons:** `assets/favicon.ico`, `assets/favicon-*.png`, `assets/apple-touch-icon.png` (generated from the logo).

## Voice

- Prefer plain language, reproducibility, and explicit data provenance.
- Name foundations and data sources; avoid implying official foundation endorsement unless true.
